lesson 05 · was ist markdown
Klartext mit Struktur
  • Einfache Textdateienmit Zeichen wie #, - und ** ** als Struktur
  • Überall lesbarEditor, Browser, GitHub — sogar als roher Text verständlich
  • Perfekt für GitKlartext lässt sich vergleichen und versionieren
▸beispiel.mdMarkdown
# Überschrift

Normaler Text — einfach lesbar.

- Listenpunkt eins
- Listenpunkt zwei

Das hier ist **fett**.
lesson 05 · warum markdown
Warum Claude Markdown liebt
  • Klartext, kein SpezialformatClaude liest und schreibt es fehlerfrei — kein Word, kein PDF nötig
  • Diff-barjede Änderung ist als Diff sichtbar — ideal fürs Review
  • VersionierbarGit speichert die Historie — nichts geht verloren
  • Struktur ohne BallastÜberschriften und Listen genügen, damit Inhalte auffindbar bleiben

lesson 05 · gute regeln
Regeln, die wirklich wirken

✕  Vage

  • „Sei vorsichtig" — vorsichtig womit?
  • „Schreib guten Code" — nicht prüfbar
  • „Denk an alles" — bedeutet: an nichts

✓  Konkret

  • „Frag nach, bevor du Dateien löschst"
  • „Nach jeder Änderung: npm test"
  • „Antworten auf Deutsch, Code-Kommentare auf Englisch"
dauerhafter kontext
Was in CLAUDE.md steht, musst du nie wieder erklären.
Jede neue Session liest die Datei automatisch — deine Regeln gelten ab der ersten Sekunde.
lesson 05 · pflege
Drei Gewohnheiten für die Pflege
  • Aktuell haltenRegeln, die nicht mehr gelten, sofort raus — veraltete Regeln stiften Verwirrung
  • Kurz haltennur was wirklich gilt — eine Seite schlägt zehn
  • Von Claude ergänzen lassen„Nimm das ab jetzt in die CLAUDE.md auf" — fertig
lesson 05 · recap
Das nimmst du mit
  • Markdown = Klartext mit Struktur — diff-bar und versionierbar
  • CLAUDE.md wird automatisch gelesen — Regeln gelten in jeder Session
  • README, PLAN, TASKS — jede Datei hat ihre eigene Rolle
  • Regeln konkret & kurz — und regelmäßig aufräumen
merksatz

„Eine Regel, einmal geschrieben — in jeder Session wirksam."
