lesson 10 · einrichtung
Schritt 2: der Eintrag in der Konfiguration
▸.mcp.jsonmcpServers · ein eintrag
// Claude-Code-Konfiguration → Abschnitt "mcpServers"
"n8n-mcp": {
  "command": "npx",
  "args": ["-y", "n8n-mcp"],
  "env": {
    "N8N_API_URL": "https://deine-instanz.n8n.cloud",
    "N8N_API_KEY": "…"
  }
}
Zwei Umgebungsvariablen genügen: N8N_API_URL (Basis-URL der Instanz) und N8N_API_KEY (dein Key).

lesson 10 · mehrere instanzen
Kunde A + Kunde B? Zwei Einträge.
▸.mcp.jsonzwei instanzen · zwei einträge
// ein Eintrag pro Instanz — eigener Name, eigener Key
"n8n-mcp-kundeA": {
  "command": "npx", "args": ["-y", "n8n-mcp"],
  "env": { "N8N_API_URL": "https://kunde-a.n8n.cloud",
           "N8N_API_KEY": "…" } },
"n8n-mcp-kundeB": {
  "command": "npx", "args": ["-y", "n8n-mcp"],
  "env": { "N8N_API_URL": "https://kunde-b.n8n.cloud",
           "N8N_API_KEY": "…" } }
Die Tools erscheinen getrennt pro Server — du sagst einfach dazu, welche Instanz du meinst.

lesson 10 · sicherheit
Mit Vollzugriff arbeitet man anders
Der API-Key ist kein Lesezugriff: Er erlaubt Vollzugriff auf die Instanz — lesen, ändern, löschen. Behandle ihn wie ein Passwort.
  • Key sicher ablegen.Passwort-Manager statt Notiz-Datei — und niemals ins Repo committen.
  • Produktiv-Instanzen mit Bedacht.Erst analysieren lassen — Änderungen bewusst und einzeln freigeben.
  • Änderungen reviewen.Bevor ein geänderter Workflow aktiv geht: einmal selbst draufschauen.
