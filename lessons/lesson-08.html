lesson 08 · schritt 2 — der mcp
Plugin rein, ein Befehl — verbunden
  • Plugin „Local REST API" installieren.Obsidian → Einstellungen → Community-Plugins → suchen, installieren, aktivieren.
  • HTTP-Server aktivieren & API-Key kopieren.Beides direkt in den Plugin-Einstellungen — der Key ist dein Zugang.
  • MCP registrieren, Claude Code neu starten.Ein Befehl (rechts) — danach stehen die Obsidian-Tools in jeder Session bereit.
terminal
❯ claude mcp add --transport http obsidian \
    http://127.0.0.1:27123/mcp/ \
    --header "Authorization: Bearer <API-KEY>"
✓ MCP-Server „obsidian" hinzugefügt

❯ /mcp
  ● obsidian — verbunden, Tools verfügbar

lesson 08 · zugangsdaten
Alles Wichtige in einer .env — wie in Lesson 07
▸obsidian.envC:\claude-zentrale\zugangsdaten\
# Obsidian Long Time Memory — Zugang & OrteOBSIDIAN_API_URL=http://127.0.0.1:27123OBSIDIAN_API_KEY=xxxxxxxxxxxxxxxx        # aus den Plugin-EinstellungenOBSIDIAN_VAULT_PATH=C:/claude-zentrale/claude-memory# Regel aus Lesson 07: referenzieren, nie ausgeben, nie committen
Ein Ort für alles Globale: Die obsidian.env liegt neben der global.env — jede Config (MCP, Skripte, neuer Rechner) bedient sich hier, statt Werte zu verstreuen.
lesson 08 · verankern
Ein Eintrag — und jedes Projekt kennt dein Gedächtnis
▸CLAUDE.mdC:\claude-zentrale\ — gilt für ALLE Projekte
## Long Time Memory (Obsidian)- Vault: C:/claude-zentrale/claude-memory/ — Zugriff via Obsidian-MCP- Zugang & Orte: zugangsdaten/obsidian.env@claude-memory/MEMORY.md   # Index wird IMMER mitgeladenRegeln: Neue dauerhafte Erkenntnis → sofort in der passenden Notizergänzen + Index aktualisieren. Veraltetes löschen, Index kompakt halten.Keine Secrets ins Memory — die liegen in der .env-Struktur.
Die Anbindung: ~/.claude/CLAUDE.md enthält nur eine Zeile — @C:/claude-zentrale/CLAUDE.md — und wird in jedem Projekt automatisch zuerst gelesen. Die README.md daneben erklärt die Struktur für Menschen.

lesson 08 · pflege
Was rein gehört — und was nicht

✕  Gehört nicht rein

  • Passwörter & API-Keys — die wohnen in der .env (Lesson 07!)
  • Tagesgeschäft & Einmaliges — das ist Chat, kein Gedächtnis
  • Alles sammeln, nie löschen — ein Museum hilft niemandem

✓  Gehört rein

  • Arbeitsweisen & Entscheidungen — inklusive dem Warum
  • Projekt- & Tool-Wissen, das nicht im Code steht
  • Pflege als Gewohnheit — nach großen Aufgaben aktualisieren, Veraltetes raus
