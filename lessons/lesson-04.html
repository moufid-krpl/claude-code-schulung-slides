warum ordnung zählt
Claude Code ist so gut wie der Ordner, in dem es arbeitet.
Der geöffnete Ordner ist der Kontext. Ordnung darin heißt: besserer Kontext — und damit bessere Ergebnisse.

Ein Ort für alles: Dev/Projekte/

Ein zentraler Arbeitsordner — nie wieder suchen, wo ein Projekt liegt.
lesson 04 · ein projekt, ein ordner
Ein Projekt = ein Ordner
  • Jedes Projekt bekommt genau einen Unterordnerklare Grenze: was drinliegt, gehört zum Projekt
  • Sprechende Namen, klein geschriebenkunde-a-shop statt „Neuer Ordner (2)"
  • Alles zum Projekt liegt im Projektkeine Dateien verstreut auf Desktop und Downloads
Wichtig: VS Code immer im Projektordner öffnen — nicht im Überordner. Sonst sieht Claude alle Projekte gleichzeitig und keines richtig.
lesson 04 · test vs. produktiv
Spielwiese und Kundenarbeit trennen

✕  Alles gemischt

  • Experimente im Kundenprojekt — „nur mal kurz testen"
  • Testdateien landen im Produktiv-Code
  • Ein Fehlgriff trifft direkt den Kunden

✓  Sauber getrennt

  • spielwiese/ für Tests, Übungen, Experimente
  • Kundenprojekte bleiben unberührt davon
  • Erst in der Spielwiese lernen, dann übertragen
lesson 04 · grundstruktur
So startet jedes neue Projekt
▸mein-projekt/Grundstruktur
mein-projekt/
  README.md     # für Menschen: was ist das Projekt?
  CLAUDE.md     # für Claude: Regeln & Arbeitsweise
  .gitignore    # was Git (und Claude) ignorieren soll
  src/          # der eigentliche Code
Vier Bausteine reichen für den Start — den Rest kann Claude für dich anlegen.
lesson 04 · häufige fehler
Vier Fehler, die dich Kontext kosten
  • Den Überordner geöffnetClaude sieht zwölf Projekte — und versteht keines richtig
  • Downloads/ als ArbeitsordnerRechnungen, Installer, Zips — Chaos als Kontext
  • Ein Riesenprojekt für allesje größer der Misch-Ordner, desto vager die Ergebnisse
  • Kopien statt Git„projekt-final-v2-NEU" — Versionierung übernimmt das Werkzeug, nicht der Explorer
lesson 04 · recap
Das nimmst du mit
  • Ein zentraler Arbeitsordner — z. B. Dev/Projekte/
  • Ein Projekt = ein Ordner — VS Code immer dort öffnen
  • Spielwiese getrennt von Kunden- und Produktivprojekten
  • Grundstruktur: README.md, CLAUDE.md, .gitignore, src/
